# 推荐算法实现说明

## 1. 实现目标

推荐系统用于根据用户求职画像，为用户推荐更匹配的贵州本地岗位，并给出推荐理由。

当前版本已经升级为：

```text
基于 TF-IDF 余弦相似度的内容推荐 + 城市/薪资/学历/经验多维规则加权
```

也就是说，岗位内容匹配分不再是简单关键词命中累加，而是通过 TF-IDF 向量和余弦相似度计算得到。

## 2. 推荐接口

```text
POST /recommend/refresh?limit=20
GET  /recommend/list?limit=20
```

两个接口都需要请求头：

```text
X-User-Id: 当前登录用户ID
```

接口说明：

```text
/recommend/refresh
重新计算当前用户的推荐岗位，删除旧推荐结果，写入新的推荐结果。

/recommend/list
查询当前用户已经生成的推荐岗位。
```

## 3. 用户画像

用户画像来自 `sys_user` 表：

```text
local_city              所在城市
education_text          学历
expected_position       期望岗位
expected_salary_min     期望最低薪资
expected_salary_max     期望最高薪资
skill_input_text        技能描述
```

用于 TF-IDF 的用户画像文本为：

```text
expected_position * 2
+ skill_input_text * 2
+ education_text
+ local_city
```

期望岗位和技能描述会重复加入一次，是为了提高用户核心意图在向量中的权重。

## 4. 岗位画像

岗位画像来自 `job_info` 表：

```text
job_title
city
education_text
experience_text
salary_min
salary_max
job_description
```

用于 TF-IDF 的岗位画像文本为：

```text
job_title * 2
+ job_description
+ education_text
+ experience_text
+ city
```

岗位标题重复加入一次，是为了让岗位名称比普通描述文本更重要。

## 5. TF-IDF 内容相似度

刷新推荐时，系统会先查询候选岗位：

```text
状态正常
岗位描述不为空
默认最多 3000 条
同城岗位优先进入候选集
```

然后以这批候选岗位作为语料库计算 TF-IDF。

处理流程：

```text
1. 对用户画像文本分词
2. 对候选岗位文本分词
3. 统计每个词在岗位语料库中的文档频率 DF
4. 计算 IDF
5. 生成用户 TF-IDF 向量
6. 生成每个岗位的 TF-IDF 向量
7. 计算用户向量与岗位向量的余弦相似度
8. 将余弦相似度换算成 0-100 的 content_score
```

IDF 公式：

```text
idf(term) = ln((N + 1) / (df(term) + 1)) + 1
```

余弦相似度：

```text
cosine(user, job) =
  dot(user_vector, job_vector)
  / (norm(user_vector) * norm(job_vector))
```

内容匹配分：

```text
content_score = cosine(user_vector, job_vector) * 100
```

## 6. 分词规则

当前 Java 实现没有引入额外中文分词库，而是使用轻量规则分词：

```text
英文、数字、技术词：按连续字母数字提取
中文文本：提取 2-4 字长度的 n-gram
```

这样可以覆盖：

```text
Java
Spring
Spring Boot
MySQL
后端
后端开发
开发工程师
```

同时会过滤一批过于通用的词，例如：

```text
岗位、职位、工作、要求、负责、经验、学历、公司、熟悉、了解、优先
```

## 7. 最终推荐分

最终推荐分采用多维加权：

```text
recommend_score =
  content_score    * 0.45
+ city_score       * 0.20
+ salary_score     * 0.15
+ education_score  * 0.10
+ experience_score * 0.10
```

各分数含义：

```text
content_score      TF-IDF 内容相似度分
city_score         城市匹配分
salary_score       薪资匹配分
education_score    学历匹配分
experience_score   经验要求友好度
```

## 8. 规则分

城市匹配：

```text
用户城市 = 岗位城市      100
用户未填写城市           60
不同城市                 35
```

薪资匹配：

```text
岗位薪资与期望薪资有重叠       100
岗位薪资略低于期望             75
岗位薪资明显低于期望           55
岗位薪资高于期望               85
用户未填写薪资                 60
岗位无薪资                     50
```

学历匹配：

```text
岗位学历不限                   90
用户学历 >= 岗位要求            100
岗位要求比用户学历高一级         65
岗位要求明显高于用户学历         45
用户未填写学历                 60
```

经验匹配：

```text
不限 / 应届生 / 无经验          90
其他经验要求                   60
```

当前用户表没有求职者工作年限字段，所以经验分第一版只判断岗位是否对应届生或无经验用户友好。

## 9. 推荐理由

推荐理由根据高分项自动生成，例如：

```text
同城岗位
TF-IDF匹配词：Java、Spring、MySQL、后端开发
薪资符合期望
学历要求符合
岗位内容与求职画像相似
```

其中“TF-IDF匹配词”来自用户向量和岗位向量共同命中的高权重词。

数据库中保存为分号分隔文本：

```text
同城岗位；TF-IDF匹配词：Java、Spring、MySQL；薪资符合期望
```

接口返回时会同时返回：

```text
reasonText    原始推荐理由文本
reasons       前端可直接渲染的推荐理由数组
```

## 10. 数据库存储

项目本地数据库已有 `recommend_result` 表，代码按现有字段写入：

```text
final_score         最终推荐分
location_score      城市匹配分
position_score      TF-IDF 内容匹配分
skill_score         TF-IDF 内容匹配分
behavior_score      学历分与经验分的平均值
salary_score        薪资匹配分
recommend_reason    推荐理由
rank_no             排名
batch_no            推荐批次
recommend_scene     固定写入 recommend
recommend_type      固定写入 content
```

如果新环境没有这张表，可以使用下面的建表 SQL：

```sql
CREATE TABLE IF NOT EXISTS recommend_result (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '推荐记录ID',
  user_id BIGINT NOT NULL COMMENT '用户ID',
  job_id BIGINT NOT NULL COMMENT '岗位ID',
  recommend_scene VARCHAR(20) NOT NULL COMMENT '推荐场景 home/recommend',
  recommend_type VARCHAR(20) NOT NULL COMMENT '推荐类型 content/behavior/hybrid',
  location_score DECIMAL(6,2) NOT NULL DEFAULT 0.00 COMMENT '地区匹配分',
  position_score DECIMAL(6,2) NOT NULL DEFAULT 0.00 COMMENT '岗位匹配分',
  skill_score DECIMAL(6,2) NOT NULL DEFAULT 0.00 COMMENT '技能或内容匹配分',
  behavior_score DECIMAL(6,2) NOT NULL DEFAULT 0.00 COMMENT '行为或扩展匹配分',
  salary_score DECIMAL(6,2) NOT NULL DEFAULT 0.00 COMMENT '薪资匹配分',
  final_score DECIMAL(6,2) NOT NULL DEFAULT 0.00 COMMENT '最终推荐分',
  rank_no INT DEFAULT NULL COMMENT '推荐排名',
  recommend_reason VARCHAR(500) DEFAULT NULL COMMENT '推荐原因说明',
  batch_no VARCHAR(50) DEFAULT NULL COMMENT '推荐批次号',
  create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  INDEX idx_user_id (user_id),
  INDEX idx_job_id (job_id),
  INDEX idx_final_score (final_score),
  INDEX idx_batch_no (batch_no),
  INDEX idx_user_scene (user_id, recommend_scene)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='推荐结果表';
```

## 11. 后端代码位置

```text
recommend/controller/RecommendController.java
recommend/service/RecommendService.java
recommend/service/impl/RecommendServiceImpl.java
recommend/mapper/RecommendMapper.java
recommend/entity/RecommendResult.java
recommend/vo/RecommendJobVO.java
recommend/support/ContentMatch.java
recommend/support/RecommendScorer.java
recommend/support/RecommendScore.java
resources/mapper/recommend/RecommendMapper.xml
```

核心流程在 `RecommendServiceImpl.refresh`：

```text
读取用户画像
-> 查询候选岗位
-> 基于候选岗位构建 TF-IDF 语料库
-> 计算用户与每个岗位的内容相似度
-> 叠加城市、薪资、学历、经验规则分
-> 按最终推荐分倒序排序
-> 取 Top N
-> 删除用户旧推荐
-> 写入 recommend_result
-> 返回推荐岗位列表
```

核心 TF-IDF 计算在 `RecommendScorer.contentMatches`。

## 12. 后续升级方向

当前版本已经是真正的 TF-IDF 内容推荐。后续可以继续优化：

```text
1. 引入 jieba、HanLP 等中文分词，提高中文岗位描述分词质量
2. 增加用户浏览、收藏、搜索行为，加入 behavior_score
3. 增加 job_skill 表，提前抽取岗位技能，减少实时文本计算
4. 将岗位 TF-IDF 向量离线预计算，刷新推荐时只计算用户向量
5. 用 Spark 离线批量计算全量用户推荐结果
6. 前端增加“为你推荐”页面，展示推荐分和推荐理由
```
