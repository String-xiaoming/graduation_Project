# 推荐算法实现说明

## 1. 实现目标

本项目第一版推荐系统采用“基于内容的岗位推荐算法”，根据用户求职画像和岗位内容特征，为用户生成可解释的岗位推荐结果。

推荐结果回答两个问题：

```text
这个用户适合哪些岗位？
为什么推荐这些岗位？
```

当前实现已接入 Spring Boot 后端，提供刷新推荐和查询推荐两个接口。

## 2. 推荐接口

```text
POST /recommend/refresh?limit=20
GET  /recommend/list?limit=20
```

两个接口都需要请求头：

```text
X-User-Id: 当前登录用户ID
```

接口说明：

```text
/recommend/refresh
重新计算当前用户的推荐岗位，删除旧推荐结果，写入新的推荐结果。

/recommend/list
查询当前用户已经生成的推荐岗位。
```

## 3. 用户画像

用户画像来自 `sys_user` 表：

```text
local_city              所在城市
education_text          学历
expected_position       期望岗位
expected_salary_min     期望最低薪资
expected_salary_max     期望最高薪资
skill_input_text        技能描述
```

第一版不依赖用户行为日志，避免推荐链路过重。用户只要完善个人资料，就可以生成推荐。

## 4. 岗位画像

岗位画像来自 `job_info` 表：

```text
job_title
company_name
city
education_text
experience_text
salary_text
salary_min
salary_max
job_description
publish_time
```

候选岗位默认从 `job_info` 中取状态正常、岗位描述不为空的最近 3000 条岗位。同城岗位会优先进入候选集。

## 5. 推荐分数

最终推荐分采用多维加权：

```text
recommend_score =
  content_score    * 0.45
+ city_score       * 0.20
+ salary_score     * 0.15
+ education_score  * 0.10
+ experience_score * 0.10
```

各分数含义：

```text
content_score      内容匹配分，来自期望岗位和技能关键词匹配
city_score         城市匹配分
salary_score       薪资匹配分
education_score    学历匹配分
experience_score   经验要求友好度
```

## 6. 内容匹配分

内容匹配使用关键词命中加权。

用户文本来源：

```text
expected_position + skill_input_text
```

岗位文本来源：

```text
job_title + job_description
```

关键词切分规则：

```text
逗号、顿号、分号、斜杠、空格、换行
```

匹配规则：

```text
期望岗位命中岗位标题：加较高分
期望岗位命中岗位全文：加中等分
技能关键词命中岗位标题：加较高分
技能关键词命中岗位描述：加普通分
最高不超过 100 分
```

这样做的好处是逻辑清晰、容易调试、推荐理由容易生成。后续如果升级，可以把 `content_score` 替换成 TF-IDF 余弦相似度。

## 7. 规则分

城市匹配：

```text
用户城市 = 岗位城市      100
用户未填写城市           60
不同城市                 35
```

薪资匹配：

```text
岗位薪资与期望薪资有重叠       100
岗位薪资略低于期望             75
岗位薪资明显低于期望           55
岗位薪资高于期望               85
用户未填写薪资                 60
岗位无薪资                     50
```

学历匹配：

```text
岗位学历不限                   90
用户学历 >= 岗位要求            100
岗位要求比用户学历高一级         65
岗位要求明显高于用户学历         45
用户未填写学历                 60
```

经验匹配：

```text
不限 / 应届生 / 无经验          90
其他经验要求                   60
```

当前用户表没有求职者工作年限字段，所以经验分第一版只判断岗位是否对应届生或无经验用户友好。

## 8. 推荐理由

推荐理由根据高分项自动生成，例如：

```text
同城岗位
匹配技能：Java、Spring、MySQL
薪资符合期望
学历要求符合
岗位方向接近期望
```

数据库中保存为分号分隔文本：

```text
同城岗位；匹配技能：Java、Spring、MySQL；薪资符合期望
```

接口返回时会同时返回：

```text
reasonText    原始推荐理由文本
reasons       前端可直接渲染的推荐理由数组
```

## 9. 数据库存储

项目本地数据库已有 `recommend_result` 表，因此代码按现有字段兼容写入：

```text
final_score         最终推荐分
location_score      城市匹配分
position_score      内容匹配分
skill_score         内容匹配分
behavior_score      学历分与经验分的平均值
salary_score        薪资匹配分
recommend_reason    推荐理由
rank_no             排名
batch_no            推荐批次
recommend_scene     固定写入 recommend
recommend_type      固定写入 content
```

如果新环境没有这张表，可以使用下面的建表 SQL：

```sql
CREATE TABLE IF NOT EXISTS recommend_result (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '推荐记录ID',
  user_id BIGINT NOT NULL COMMENT '用户ID',
  job_id BIGINT NOT NULL COMMENT '岗位ID',
  recommend_scene VARCHAR(20) NOT NULL COMMENT '推荐场景 home/recommend',
  recommend_type VARCHAR(20) NOT NULL COMMENT '推荐类型 content/behavior/hybrid',
  location_score DECIMAL(6,2) NOT NULL DEFAULT 0.00 COMMENT '地区匹配分',
  position_score DECIMAL(6,2) NOT NULL DEFAULT 0.00 COMMENT '岗位匹配分',
  skill_score DECIMAL(6,2) NOT NULL DEFAULT 0.00 COMMENT '技能匹配分',
  behavior_score DECIMAL(6,2) NOT NULL DEFAULT 0.00 COMMENT '行为或扩展匹配分',
  salary_score DECIMAL(6,2) NOT NULL DEFAULT 0.00 COMMENT '薪资匹配分',
  final_score DECIMAL(6,2) NOT NULL DEFAULT 0.00 COMMENT '最终推荐分',
  rank_no INT DEFAULT NULL COMMENT '推荐排名',
  recommend_reason VARCHAR(500) DEFAULT NULL COMMENT '推荐原因说明',
  batch_no VARCHAR(50) DEFAULT NULL COMMENT '推荐批次号',
  create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  INDEX idx_user_id (user_id),
  INDEX idx_job_id (job_id),
  INDEX idx_final_score (final_score),
  INDEX idx_batch_no (batch_no),
  INDEX idx_user_scene (user_id, recommend_scene)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='推荐结果表';
```

## 10. 后端代码位置

```text
recommend/controller/RecommendController.java
recommend/service/RecommendService.java
recommend/service/impl/RecommendServiceImpl.java
recommend/mapper/RecommendMapper.java
recommend/entity/RecommendResult.java
recommend/vo/RecommendJobVO.java
recommend/support/RecommendScorer.java
recommend/support/RecommendScore.java
resources/mapper/recommend/RecommendMapper.xml
```

核心流程在 `RecommendServiceImpl.refresh`：

```text
读取用户画像
-> 查询候选岗位
-> 逐个岗位计算推荐分
-> 按推荐分倒序排序
-> 取 Top N
-> 删除用户旧推荐
-> 写入 recommend_result
-> 返回推荐岗位列表
```

核心打分在 `RecommendScorer.score`。

## 11. 后续升级方向

第一版是轻量内容推荐，已经可以用于毕业设计展示。后续可以升级：

```text
1. 将内容匹配从关键词命中升级为 TF-IDF 余弦相似度
2. 新增用户浏览、收藏、搜索行为，加入 behavior_score
3. 增加 job_skill 表，提前抽取岗位技能，减少实时文本匹配
4. 用 Spark 离线批量计算全量用户推荐结果
5. 前端增加“为你推荐”页面，展示推荐分和推荐理由
```
